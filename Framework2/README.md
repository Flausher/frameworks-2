# Framework2

Проект Framework2 представляет модульное приложение на платформе .NET. Программа подгружает плагины из указанной папки. Система выстраивает правильную последовательность старта модулей на основе их зависимостей. Контейнер внедрения зависимостей управляет жизненным циклом сервисов.

## Основные компоненты приложения

1. Загрузчик настроек читает файл appsettings.json. Файл содержит путь к директории плагинов. Там же находится список разрешенных к запуску компонентов.
2. Сканер ищет скомпилированные библиотеки в целевой папке. Он фильтрует классы по интерфейсу IAppModule.
3. Загрузчик модулей анализирует связи между компонентами. Ядро строит граф зависимостей. Алгоритм топологической сортировки определяет правильный порядок запуска. Программа прерывает работу при нахождении циклических ссылок.

## Практическая реализация

Каждый модуль реализует контракты интерфейса IAppModule. Пакет обязан передать имя компонента и перечень зависимостей.

Модуль логирования регистрирует сервис AuditTrail. Данный класс сохраняет текстовые сообщения в память программы.
Компонент валидации фильтрует числовые значения. Он удаляет числа вне диапазона от 0 до 100.
Модуль отчетов вычисляет среднее арифметическое значение массива. Вы увидите итоговый результат на экране терминала.

## Сборка и запуск

Откройте терминал в корневой папке решения.
Введите команду dotnet build для компиляции исходного кода.
Перейдите в папку Framework2.
Введите команду dotnet run для выполнения программы.

## Инструкции по тестированию

Вы найдете сценарии проверок в файле ModuleBootstrapperTests.cs. Тесты подтверждают правильность сортировки зависимостей. Они контролируют реакцию системы на ошибки конфигурации.

Перейдите в корневую директорию решения.
Введите команду dotnet test для старта тестирования.
Терминал выведет результаты прохождения всех проверок.

## Пошаговый запуск приложения

1. Откройте терминал.
2. Перейдите в корневую директорию вашего проекта.
3. Выполните сборку решения командой `dotnet build`. В этот момент скопилируются все модули.
4. Откройте папку Framework2 через терминал.
5. Скопируйте нужные dll-файлы модулей в папку `bin/Debug/net10.0` (если они отсутствуют).
6. Выполните команду `dotnet run`.
7. Проверьте вывод в консоли. Там отобразится результат инициализации и работы модулей.
